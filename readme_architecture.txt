이 코드베이스는 여러 부분으로 구성된 모던 웹 애플리케이션 아키텍처를 가지고 있습니다. 각 디렉토리의 역할을 중심으로 설명해 드리겠습니다.

### 전체 아키텍처 요약

이 프로젝트는 **모노레포(Monorepo)** 구조로, 하나의 저장소 안에 프론트엔드, 백엔드, 데이터베이스 설정 등 여러 하위 프로젝트가 함께 관리되고 있습니다.

- **프론트엔드 (Frontend)**: 사용자에게 보여지는 웹 페이지를 담당합니다.
- **백엔드 (Backend)**: 서버 로직, 데이터 처리, API 제공을 담당합니다.
- **데이터베이스 (Database)**: 데이터를 저장하고 관리합니다.
- **도커 (Docker)**: 개발 및 배포 환경을 통일하기 위해 컨테이너 기술을 사용합니다.

---

### 주요 디렉토리 설명

1.  **`frontend/` (프론트엔드)**
    - **역할**: 사용자가 직접 상호작용하는 웹 애플리케이션의 UI(사용자 인터페이스)를 담당합니다.
    - **기술 스택**:
        - **React, TypeScript**: UI를 구축하기 위한 자바스크립트 라이브러리와 타입스크립트를 사용합니다.
        - **Vite**: 빠르고 효율적인 개발 서버 및 빌드 도구입니다.
        - **Tailwind CSS**: CSS 프레임워크로, 신속하게 UI를 디자인합니다.
    - **주요 파일/디렉토리**:
        - `src/pages/`: `Dashboard.tsx`, `Artists.tsx` 등 각 페이지를 구성하는 컴포넌트가 있습니다.
        - `src/components/`: `Header.tsx`, `Sidebar.tsx`처럼 여러 페이지에서 재사용되는 공통 UI 조각들이 있습니다.
        - `package.json`: 프로젝트에 필요한 라이브러리 의존성을 관리합니다.

2.  **`backend/` (백엔드)**
    - **역할**: 애플리케이션의 핵심 로직을 처리하고, 프론트엔드에 필요한 데이터(API)를 제공하며, 데이터베이스와 통신합니다.
    - **기술 스택**:
        - **Python, Flask**: 웹 프레임워크인 Flask를 사용하여 API 서버를 구축합니다.
        - **SQLAlchemy, Flask-Migrate**: 데이터베이스 작업을 쉽게 처리하고(ORM), 데이터베이스 스키마 변경을 관리합니다.
    - **주요 파일/디렉토리**:
        - `routes/`: `artists.py`, `news.py` 등 API 엔드포인트(경로)를 정의합니다.
        - `models/`: `artist.py`, `news.py` 등 데이터베이스 테이블 구조를 파이썬 클래스로 정의합니다.
        - `services/`: `news_crawler.py` (뉴스 크롤링), `scheduler.py` (주기적인 작업 스케줄링) 등 특정 비즈니스 로직을 수행하는 서비스가 있습니다.
        - `requirements.txt`: 필요한 파이썬 패키지 목록입니다.

3.  **`database/` (데이터베이스)**
    - **역할**: 데이터베이스의 초기 구조(스키마)와 초기 데이터를 관리합니다.
    - **주요 파일/디렉토리**:
        - `init.sql`: 프로젝트 시작 시 필요한 데이터베이스 테이블을 생성하는 SQL 스크립트입니다.
        - `migrations/`: 데이터베이스 구조가 변경될 때마다 그 이력을 관리하는 파일들이 저장됩니다. (Alembic/Flask-Migrate 사용)
        - `seeds/`: 개발 초기 단계에서 필요한 기본 데이터를 미리 넣어두는 스크립트가 위치할 수 있습니다.

4.  **`docker/` (도커)**
    - **역할**: 프론트엔드, 백엔드, 데이터베이스를 각각 독립된 환경(컨테이너)에서 실행하고 서로 연결하여, 어떤 개발 환경에서든 동일하게 애플리케이션을 실행할 수 있도록 합니다.
    - **주요 파일**:
        - `docker-compose.yml`: 여러 컨테이너(예: 웹서버, API서버, DB)를 한 번에 정의하고 실행하기 위한 설정 파일입니다.

5.  **`scripts/` (스크립트)**
    - **역할**: `start_backend.sh`, `start_frontend.sh` 와 같이 개발 과정에서 반복적으로 사용하는 명령어들을 셸 스크립트로 만들어 개발 편의성을 높입니다.

### 동작 흐름

1.  사용자는 **프론트엔드** 웹 페이지에 접속합니다.
2.  프론트엔드는 필요한 데이터를 **백엔드**의 API 서버에 요청합니다.
3.  백엔드는 요청을 받아 비즈니스 로직을 수행하고, 필요시 **데이터베이스**에서 데이터를 조회하거나 수정합니다.
4.  백엔드는 처리 결과를 프론트엔드에 응답으로 보내주고, 프론트엔드는 이 데이터를 화면에 표시합니다.
5.  이와 별개로, `scheduler.py` 같은 서비스는 주기적으로 `news_crawler.py`를 실행하여 외부 뉴스를 수집하고 데이터베이스에 저장하는 등의 백그라운드 작업을 수행할 수 있습니다.
